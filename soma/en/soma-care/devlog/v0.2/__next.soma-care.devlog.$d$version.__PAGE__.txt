4:T153b,<h1 id="v02--replacing-mock-skills-with-a-truly-autonomous-stretch">V0.2 — Replacing mock skills with a truly autonomous Stretch</h1>
<p>v0.1 proved the ANIMA cognition stack could close the loop under language input. v0.2 replaces the mock skills underneath with <strong>genuinely autonomous execution</strong> — every run queries object poses from the MJCF, solves IK, and steps physics. No script replay.</p>
<p>A pivot happened during implementation, worth recording.</p>
<h2 id="pivot-from-ros-2--gazebo-to-pure-python-mujoco">Pivot: from ROS 2 + Gazebo to pure-Python MuJoCo</h2>
<p><strong>Original plan</strong>: Docker + ROS 2 Humble + Gazebo Harmonic + MoveIt 2 + Nav2 — the canonical "heavy" robot stack.</p>
<p>Three problems surfaced during execution:</p>
<ol>
<li><code>hello-robot</code> does not ship a <code>stretch_moveit2</code> apt package or GitHub config. We would have had to write the SRDF + kinematics config ourselves (3–5 hours of work)</li>
<li>The <code>stretch_mujoco</code> Python API was already sufficient: <code>sim.pull_camera_data()</code> → RGB frames; <code>sim.move_to(Actuators.lift, pos)</code> + <code>wait_until_at_setpoint</code> → joint position control; <code>sim.set_base_velocity(v, ω)</code> → base velocity. The Stretch arm only has 5 DoF (lift Z + arm extension + wrist yaw/pitch/roll) — analytical IK fits in 10 lines</li>
<li>Nav2 is visibly overkill for a closed single-room + hallway scene</li>
</ol>
<p><strong>New plan</strong>: pure-Python <code>stretch_mujoco</code> API + analytical IK + unicycle PID. ANIMA's L3 skill calls <code>stretch_mujoco</code> directly, no ROS nodes.</p>
<p>Side benefits that made this the right call:</p>
<ul>
<li>MuJoCo supports macOS natively, so the laptop can run the sim without VSCode Remote SSH</li>
<li>Deployment collapses to three processes (FastAPI + Next.js + MuJoCo); no rosdep / colcon / DDS tuning</li>
<li>The real-hardware path isn't closed: v1.x can swap <code>SimSkillBehaviour</code> for <code>Ros2SkillBehaviour</code> while leaving the <code>py_trees</code> orchestration untouched</li>
</ul>
<p>The ROS 2 Humble + stretch_ros2 workspace stays around as a reference for future hardware work.</p>
<h2 id="six-l3-skill-primitives-wired-up">Six L3 skill primitives wired up</h2>
<p>A new <code>SimSkillBehaviour</code> base class plus six concrete skills, all sharing one blackboard dict:</p>
<ul>
<li><code>locate</code> — query the target body's pose in simulation</li>
<li><code>navigate</code> — unicycle PID with align / approach hysteresis to prevent oscillation</li>
<li><code>grasp</code> — analytical IK + close gripper</li>
<li><code>lift</code> — raise to a safe height</li>
<li><code>deliver</code> — drive to the bedside pose</li>
<li><code>release</code> — open gripper</li>
</ul>
<p>L2 Planner also gains a "fall back to v0.1 mock skills if simulation unavailable" branch — so local zero-config still runs end to end.</p>
<h2 id="simulation-view-lands-in-the-frontend">Simulation view lands in the frontend</h2>
<p>Backend <code>sim/manager.py</code> holds the <code>StretchMujocoSimulator</code> and renders a <code>demo_view</code> third-person camera itself. The <code>stretch_mujoco</code> camera pipeline doesn't expose custom MJCF cameras, so this path bypasses it via <code>mujoco.Renderer</code> directly.</p>
<p>Three routes:</p>
<ul>
<li><code>/api/sim/mjpeg</code> — multipart MJPEG stream</li>
<li><code>/api/sim/reset</code> — reset scene to initial state</li>
<li><code>/api/sim/status</code> — health probe</li>
</ul>
<p>Frontend adds a <code>SimulationView.tsx</code> component (an <code>&#x3C;img></code> fed by the MJPEG stream plus a Reset button). Layout expands from three columns to four (nav / sim live / intent+BT / factors).</p>
<h2 id="l5-starts-being-data-driven">L5 starts being data-driven</h2>
<p>In <code>l5_assessment.py</code>, <code>p_skill</code> moves from a fixed 0.91 constant to a rolling success-rate read from <code>pea_log.jsonl</code>. PEA finally starts feeding back into GOA instead of being a static number.</p>
<h2 id="deployment">Deployment</h2>
<p>Hetzner (<code>89.167.35.145</code>), three systemd units + nginx reverse proxy: <code>/</code> → Next.js prod, <code>/api</code> → FastAPI, <code>/ws</code> → FastAPI WebSocket upgrade. Nginx buffering is disabled for <code>/api/sim/</code> so MJPEG streams in real time.</p>
<h2 id="acceptance-end-to-end-script">Acceptance (end-to-end script)</h2>
<ul>
<li>Open <code>http://89.167.35.145</code>, see the ward scene (bed, nightstand, water cup, Stretch at starting pose)</li>
<li>Type "I want water" → L0 through L5 light up sequentially → TaskSpec emits <code>DRINK_WATER</code> → behavior-tree six nodes turn green</li>
<li>MJPEG shows Stretch <strong>autonomously</strong> navigating, arming toward the cup, closing the gripper, lifting, returning to bedside — each run recomputes IK, <strong>not replay</strong></li>
<li>Five factors update as the task progresses; PEA logs a new <code>outcome=success</code> row</li>
<li>The Reset Sim button returns the scene to initial state within 2 seconds</li>
</ul>
