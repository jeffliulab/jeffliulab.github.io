4:T1ac8,<h1 id="anima-v010--首个参考实现落地">ANIMA V0.1.0 — 首个参考实现落地</h1>
<p>过去两周，ANIMA 这个独立仓一直是个占位骨架：一个 7 行的 Python 文件，<code>__version__ = "0.0.0"</code>，模块边界画好了但真正可运行的代码几乎没有。同一时间，在一个 BCI 医疗护理研究原型（<code>human-brain-interface-demo</code>）里，L0–L5 六层 + 五因素 + LLM-as-Parser 的一套参考实现长了出来——11 个文件、1037 行代码、pytest 全绿、pip 可装。</p>
<p><strong>2026-04-21 把这两件事合了</strong>：参考实现从原型仓搬进 ANIMA 主仓，剥离 BCI 专属术语后通用化，版本从 0.0.0 直接跳到 <strong>v0.1.0</strong>。</p>
<h2 id="六层-l0l5-全部有代码">六层 L0–L5 全部有代码</h2>
<pre><code>   上游信号（BCI / 语音 / 视觉 / 文本）
            ↓
   ┌─────────────────────────────┐
   │ L0 Signal                   │  signal → intent token + confidence + drift
   ├─────────────────────────────┤
   │ L1 Parser (LLM)             │  instruction → TaskSpec（强制 tool-calling）
   ├─────────────────────────────┤
   │ Test-and-Check（六门守卫）    │  JSON / intent / skill / params / safety / preconds
   ├─────────────────────────────┤
   │ L2 Planner                  │  TaskSpec → py_trees 行为树
   ├─────────────────────────────┤
   │ L3 Skill                    │  Function-Calling + Affordance Scoring
   ├─────────────────────────────┤
   │ L4 Adapter                  │  设备无关执行（机械臂 / 底盘 / ...）
   ├─────────────────────────────┤
   │ L5 Assessment               │  ITA · MQA · SQA · GOA · PEA（Pre / Runtime / Post）
   └─────────────────────────────┘
</code></pre>
<p>每一层都有实际源码文件（<code>src/anima/l0_signal.py</code> 到 <code>src/anima/l5_assessment.py</code>），不再是 docstring 占位。</p>
<h2 id="五因素公式落地">五因素公式落地</h2>
<p>v0.0.0 只定义了五因素的名字；v0.1.0 给了它们具体公式：</p>
<ul>
<li><strong>ITA</strong> 意图对齐度，吸收上游 drift：<code>MQA_ita = 1 − drift_score</code></li>
<li><strong>MQA</strong> 动作质量：轨迹平滑度 / 接触力包络 / 抓取余量</li>
<li><strong>SQA</strong> 技能胜任度：Beta 先验从历史 <code>pea_log.jsonl</code> 滑窗更新</li>
<li><strong>GOA</strong> 目标达成：乘法组合 <code>P(success) = ∏ Pᵢ</code>，<strong>不做平均</strong></li>
<li><strong>PEA</strong> 偏好-经验检索：三因素加权 <code>recency × 0.5 + relevance × 3.0 + importance × 2.0</code></li>
</ul>
<p>这些不是占位符——是从研究原型里跑通过的实现。</p>
<h2 id="llm-as-parserllm-agnostic">LLM-as-Parser，LLM-agnostic</h2>
<p>L1 通过 <code>LLMToolCaller</code> 协议接 LLM，不绑任何厂商。当前参考实现同时支持 DeepSeek 和 OpenAI（100% OpenAI SDK 兼容）。核心约束是<strong>强制 tool-calling</strong>——LLM 只能输出结构化 TaskSpec JSON，不能自由生成低层命令。</p>
<h2 id="剥离-bci-独有术语">剥离 BCI 独有术语</h2>
<p>原始参考实现是从一个 BCI 研究原型里长出来的，所以里面带不少 BCI 独有语义。这次合进主仓时把这些通用化：</p>
<ul>
<li><code>drift_score</code> 不再专指神经信号漂移，docstring 改成"upstream signal uncertainty, e.g. BCI drift or ASR confidence"</li>
<li><code>IntentToken</code> 的 35 词词表保留，但标注"example vocabulary; extend per application"</li>
<li>README / docstring 里的 "BCI" / "neural" / "patient" / "clinician" 字样换成通用语言</li>
<li><code>EmbodiedAdapter</code> 协议本来就通用，这次在 README 里说清楚能接 manipulator / mobile base / wheelchair / humanoid</li>
</ul>
<p>关键词从 <code>bci</code> / <code>assistive-robotics</code> 换成 <code>cognitive-architecture</code> / <code>embodied-ai</code> / <code>behavior-tree</code> / <code>llm-tool-use</code> / <code>intent-to-action</code> / <code>self-assessment</code>。</p>
<h2 id="开源脚手架就位">开源脚手架就位</h2>
<p>之前只有 LICENSE 和 README。v0.1.0 带来了：</p>
<ul>
<li><code>pyproject.toml</code> — <code>pip install -e .</code> 可装</li>
<li><code>tests/test_pipeline.py</code> — 6/6 绿</li>
<li><code>CONTRIBUTING.md</code> / <code>SECURITY.md</code> / <code>CODE_OF_CONDUCT.md</code> / <code>CHANGELOG.md</code></li>
<li>PR 模板</li>
</ul>
<p>安装方式：</p>
<pre><code class="language-bash">pip install git+https://github.com/jeffliulab/anima.git@main
</code></pre>
<p>下游应用（soma-care）就是这样依赖的，不再内嵌副本。</p>
